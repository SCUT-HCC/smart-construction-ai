## 4. Agent 调用流程

```
Step 1  从 Ch2（工程概况）提取工程信息
        ↓
Step 2  按 ch06_施工方法.md Step 1 决策树分类
        ↓
Step 3  加载本目录对应模板 → 确定 6.1~6.N 子章节结构
        ├─ 裁剪【如适用】小节（按实际工程子类型）
        └─ 混合工程 → 加载多个模板，每类独立成节
        ↓
Step 4  从 process_references 对应文件提取技术参数 → 填入模板槽位
        ↓
Step 5  从 fragments.jsonl 检索同类型高密度片段 → 参考措辞风格
        ↓
Step 6  执行跨章节一致性校验（模板 §5 检查清单）
```

## 5. 统一格式规范

### 5.1 模板文件结构

每个模板必须包含以下 6 个标准段落：

| 段落 | 内容 |
|------|------|
| §1 模板使用说明 | 适用范围、子类型选择、调用步骤 |
| §2 施工工艺总流程（6.1） | 文本流程图 + 说明文字 |
| §3 分项工程模板（6.2~6.N-1） | 每个分项的 5 段标准内容 |
| §4 验收标准汇总（6.N） | 统一格式验收表格 |
| §5 跨章节一致性检查清单 | 6 项强制检查 |
| §6 附录：完整输出示例 | 基于真实片段组装的完整章节示例 |

### 5.2 分项工程 5 段标准结构

```markdown
### X.Y {{分项名称}}

#### X.Y.1 施工工艺流程
（文本流程图）

#### X.Y.2 施工方法
（编号步骤列表 + 操作要求）

#### X.Y.3 关键技术参数
（参数/标准值/依据 三列表格）

#### X.Y.4 质量控制要点
（质控检查项列表）

#### X.Y.5 注意事项
（季节性措施、跨章节提醒等）
```

### 5.3 标记语法

| 标记 | 含义 |
|------|------|
| `{{槽位名}}` | Agent 必须填充的内容 |
| `【如适用】` | 根据工程子类型决定是否包含 |
| `> 参数来源：xxx.md §Y` | 标注参数的提取位置 |
