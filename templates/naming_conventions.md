# 施工方案章节命名规范

> **适用范围**: 南方电网施工方案编制与审核
> **制定依据**: 基于16份实际施工方案分析（详见 `docs/analysis/`）
> **版本**: v1.0 | 2026-02-20

---

## 一、章节编号规范

### 1.1 一级标题（章）

**推荐方式**: 中文数字 + 顿号 + 章节名称

```markdown
✅ 正确示例：
## 一、编制依据
## 二、工程概况
## 三、施工组织机构及职责
```

**禁止方式**:
```markdown
❌ 错误示例：
## 第一章 编制依据        # 冗余，禁止使用"第X章"
## 1. 编制依据             # 混用阿拉伯数字
## 1 编制依据              # 编号格式不统一
## 第1章 编制依据          # 混用中文和阿拉伯数字
```

---

### 1.2 二级标题（节）

**推荐方式**: 章节号.节号 + 空格 + 节名称

```markdown
✅ 正确示例：
### 2.1 工程简介
### 2.2 工程规模
### 2.3 工程地质及地貌状况
```

**禁止方式**:
```markdown
❌ 错误示例：
### （一）工程简介         # 与一级标题编号混淆
### 2-1 工程简介           # 使用短横线
### 2.1、工程简介         # 多余的顿号
```

---

### 1.3 三级标题（条）

**推荐方式**: 章节号.节号.条号 + 空格 + 条名称

```markdown
✅ 正确示例：
#### 7.2.1 质量保证体系
#### 7.2.2 过程质量控制
```

### 2.2 二级标题命名规范

**原则**:
- 使用规范、简洁的名称
- 避免冗长描述性标题
- 同类章节命名风格保持一致

**示例**:

✅ **正确命名**:
```markdown
### 2.1 工程简介
### 2.2 工程规模
### 2.3 工程地质及地貌状况
### 2.4 参建单位
```

❌ **错误命名**:
```markdown
### 2.1 工程简介及基本情况说明        # 冗长
### 2.2 本工程的规模                  # 多余的"本工程"
### 2.3 地质情况                      # 过于简略
### 2.4 建设单位、设计单位、施工单位  # 应简化为"参建单位"
```

## 五、命名规范执行流程

### 5.1 编写阶段

1. **模板选择**: 使用 `templates/standard_50502.md` 作为起点
2. **章节核对**: 对照本规范进行章节命名
3. **自查**: 使用第四部分的检查清单进行自查

### 5.2 审核阶段

1. **命名检查**: 对照标准名称表（第2.1节）逐一核对
2. **编号检查**: 确认编号格式符合规范（第一部分）
3. **完整性检查**: 使用检查清单确认必备章节齐全

### 5.3 智能检测（未来）

计划开发自动化检测工具：
- 章节名称自动校验
- 编号格式自动检查
- 完整性自动评分

## 六、历史遗留问题处理

### 6.1 已有方案的命名修正

对于已编写的施工方案，如章节命名不符合本规范：

**修正原则**:
1. **新方案**: 严格按照本规范执行
2. **修订方案**: 建议按本规范调整章节命名
3. **历史方案**: 仅做记录，不强制修改

### 6.2 过渡期处理

**过渡期**: 2026-02-20 至 2026-06-30（中期检查前）

**要求**:
- 新编方案必须符合本规范
- 修订方案建议调整
- 历史方案允许保留原命名

### 7.2 常见问题 FAQ

**Q1: 为什么不使用"第一章"格式？**
A1: 分析发现"第X章"格式冗余，且在实际文档中使用率低。中文数字 + 顿号更简洁。

**Q2: 可以使用阿拉伯数字编号吗？**
A2: 不推荐。为保持一致性，统一使用中文数字编号一级标题。

**Q3: 必须严格使用标准名称吗？**
A3: 是的。统一命名有助于自动化检测和智能生成系统的开发。

**Q4: "绿色施工与环境保护"章节是必选吗？**
A4: 当前为可选，但建议包含。该内容符合国家绿色发展战略。

**Q5: 旧方案需要修改命名吗？**
A5: 新方案必须符合规范，修订方案建议调整，历史方案允许保留原命名。
